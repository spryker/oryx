### BREAKING CHANGES

* Form controls, such as `input`, `select` or `textarea`, are now mandatory. Whenever the control is not available, and error will be thrown. This makes our APIs cleaner, as we no longer need to use conditionals whenever we query for the form control. There's still a fallback `input` form control available in the main slot. 
* - the `filter` property is no longer used, as we rely on the `filterStrategy`.
- the `filterStrategy` enum has now string values rather than number:
  - `0` became `start_with`
  - `1` became `start_of_word`
  - `2` became `contains`
* - we only support `oryx-option` components inside the popover, so that we can select options inside a complex layouts. 
* move LIT elements to 2.1
